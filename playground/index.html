moholy.js is a minimalist signal-to-shader pipeline. It connects any source of numbers — a microphone, a WebSocket feed, a gyroscope, a clock — to the visual parameters of a WebGL shader.

The library handles normalization, smoothing, palette lookup, and output formatting. You handle the rendering. It works with Three.js, raw WebGL, or anything that accepts uniform values.

Named after Laszlo Moholy-Nagy (1895–1946), who built the Light-Space Modulator — a machine that transformed light into kinetic visual patterns.

I.

Signals

A signal is a source of numbers that change over time. Each signal exposes a Float32Array of channel values, normalized to 0–1.

Six built-in sources ship with moholy. All share the same interface: .connect(), .disconnect(), .update(dt).

SourceConstructorChannels
Audiosignal.audio()FFT bins
WebSocketsignal.websocket(url)Parsed values
Fetchsignal.fetch(url)Polled JSON
Arraysignal.array(data)Manual
Devicesignal.device()Gyro + Accel
Clocksignal.clock()Oscillators
const sig = signal.clock({ period: 3 }) sig.connect() // In your render loop: sig.update(dt) // sig.data[0] = elapsed (0-1) // sig.data[2] = sin oscillator // sig.data[3] = cos oscillator

II.

Encodings

An encoding maps a signal channel to a visual dimension. You declare what each channel controls — color, displacement, opacity, emission — and moholy handles the math.

The output is plain numbers and RGB values. Plug them into any shader uniform, any material property, any animation parameter.

const vis = encode(sig, { color: { channel: 2, palette: 'spectral' }, displacement: { channel: 3, range: [-0.4, 0.4] }, emission: { channel: 4, range: [0, 1.5] }, }) // Each frame: vis.update(dt) vis.uniforms.color // { r, g, b } vis.uniforms.displacement // number vis.uniforms.emission // number
spectral palette — clock driven
III.

Palettes

Five built-in palettes map a 0–1 value to a color. Pass a palette name or your own array of hex strings.

Palette sampling is continuous — moholy interpolates between color stops, not nearest-neighbor. The result is smooth gradients across the full range.

// Built-in { palette: 'thermal' } // Custom { palette: ['#000', '#ff0066', '#00ffcc', '#fff'] }
IV.

Transforms

Transforms modify the signal value before it reaches the visual output. They compose: smooth the raw data, threshold it, apply an easing curve, then remap the range.

TransformEffect
smooth: 0.1Exponential moving average. Higher = more lag.
threshold: 0.4Values below cutoff become 0.
invert: trueFlip: 1 - value.
curve: 'ease-in'Easing: linear, ease-in, ease-out, ease-in-out.
range: [-0.5, 0.5]Remap 0–1 to any output range.
smooth vs raw — same signal
V.

GLSL Chunks

Four optional shader function libraries. Import as strings, concatenate into your shader source. Each chunk is self-contained — no dependencies between them.

ChunkFunctions
glsl.colormoholyColor(), moholyColorBlend()
glsl.displacemoholyDisplace(), moholyDisplaceNoisy(), moholyDisplaceRadial()
glsl.emissionmoholyEmission(), moholyEmissionPulse(), moholyRimEmission()
glsl.noisemoholyNoise2D(), moholyNoise3D(), moholyFBM()
import { glsl } from 'moholy' const frag = glsl.noise + glsl.color + ` void main() { float n = moholyNoise2D(vUv * 10.0); vec3 col = moholyColor(n, uPalette); gl_FragColor = vec4(col, 1.0); } `
FBM noise — 4 octaves
